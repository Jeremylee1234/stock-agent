## 测试概述

系统实现了完整的 SSE (Server-Sent Events) 流式输出功能，支持以下事件类型：

### 事件类型列表

1. **workflow_start** - 工作流开始
2. **stage_start** - 阶段开始
3. **stage_progress** - 阶段进度
4. **tool_call** - 工具调用
5. **tool_result** - 工具结果
6. **analysis_chunk** - 分析内容增量
7. **stage_complete** - 阶段完成
8. **final_answer** - 最终答案
9. **error** - 错误事件
10. **workflow_complete** - 工作流完成

### 工作流阶段

系统包含 4 个主要阶段：

1. **analyze_question** - 分析用户问题
2. **collect_data** - 收集数据
3. **analyze_data** - 分析数据
4. **generate_answer** - 生成答案

## 测试方式

提供了 3 种测试方式：

### 方式 1: 直接测试 (推荐用于开发调试)

直接测试 `StockAnalysisGraph` 的 `astream_with_events` 方法，无需启动 API 服务。

**优点:**
- 无需启动 API 服务
- 可以直接调试工作流代码
- 输出详细的事件信息

**缺点:**
- 不测试 HTTP/SSE 层
- 不测试前后端集成

### 方式 2: SSE 接口测试 (推荐用于集成测试)

通过 HTTP 客户端测试 SSE 接口，模拟真实的前后端通信。

**优点:**
- 测试完整的 HTTP/SSE 流程
- 验证 API 接口正确性
- 接近生产环境

**缺点:**
- 需要启动 API 服务
- 需要安装 httpx 库

### 方式 3: 前端测试 (推荐用于用户体验测试)

使用浏览器测试前端 EventSource 连接，验证用户界面。

**优点:**
- 测试真实的用户体验
- 可视化事件流
- 验证前端集成

**缺点:**
- 需要启动 API 服务
- 需要浏览器环境

1. **生成前端测试文件**


## 故障排查

### 问题 1: 无法连接到 API 服务

**症状:**
```
✗ 连接失败: 无法连接到服务器
```

**解决方法:**
1. 确认 API 服务已启动: `python api/main.py`
2. 检查端口是否正确（默认 8000）
3. 检查防火墙设置

### 问题 2: 工具未加载

**症状:**
```
✗ 警告: 无法导入 stock_analysis_tool 中的工具
```

**解决方法:**
1. 检查 `tools/stock_analysis_tool.py` 文件是否存在
2. 检查依赖是否安装: `pip install -r requirements.txt`
3. 检查环境变量配置（API 密钥等）

### 问题 3: SSE 连接中断

**症状:**
- 事件流突然停止
- 收到错误事件

**解决方法:**
1. 检查网络连接
2. 检查 API 服务日志
3. 增加超时时间配置
4. 检查工具调用是否失败

### 问题 4: 前端无法接收事件

**症状:**
- 浏览器控制台显示 CORS 错误
- 事件列表为空

**解决方法:**
1. 确认 API 服务已配置 CORS
2. 检查浏览器控制台错误信息
3. 确认 API 端点地址正确
4. 尝试使用 Chrome/Firefox 最新版本

### 问题 5: 数据压缩相关错误

**症状:**
```
✗ 压缩工具输出失败
```

**解决方法:**
1. 检查 `data_cache` 目录是否存在且可写
2. 检查磁盘空间
3. 查看详细错误日志

## 性能测试

### 测试指标

1. **事件延迟**: 从工作流产生事件到前端接收的时间
2. **总耗时**: 完整查询的处理时间
3. **事件数量**: 单次查询产生的事件总数
4. **内存使用**: 工作流执行期间的内存占用

### 性能基准

- 简单查询（单股票基本信息）: 5-10 秒
- 中等查询（单股票完整分析）: 10-20 秒
- 复杂查询（多股票对比分析）: 20-40 秒
- 历史模式搜索: 30-60 秒

### 优化建议

1. **启用数据压缩**: 减少大数据传输
2. **调整消息历史长度**: 避免上下文溢出
3. **配置并发限制**: 防止资源耗尽
4. **使用缓存**: 减少重复 API 调用

## 高级测试

### 并发测试

```python
import asyncio

async def concurrent_test():
    """并发测试多个查询"""
    from agents.stock_agent_main import StockAnalysisGraph
    
    graph = StockAnalysisGraph()
    
    queries = [
        "分析贵州茅台",
        "分析中国平安",
        "分析招商银行"
    ]
    
    tasks = []
    for i, query in enumerate(queries):
        task = graph.ainvoke(
            query=query,
            config={"configurable": {"thread_id": f"concurrent_{i}"}}
        )
        tasks.append(task)
    
    results = await asyncio.gather(*tasks)
    print(f"完成 {len(results)} 个并发查询")

asyncio.run(concurrent_test())
```
